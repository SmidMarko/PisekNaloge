Présentation

Fonctionnalités

Cette petite librairie C++ a pour but d'aider à la création de fichiers tests en permettant de générer des labyrinthes avec des murs, comme celui illustré ci-dessous :

Elle propose de nombreux procédés de génération, permettant d'obtenir des types de labyrinthes différents afin de créer des fichiers tests variés.

Voici la liste des fonctions proposées (les détails sont fournis plus bas sur cette page) :

  • Pour la génération de labyrinthes aléatoires acycliques et connexes :
    • vector< vector< int > > rand_maze(int nbLines, int nbCols);
    • vector< vector< int > > rand_rec_split_maze(int nbLines, int nbCols);
    • vector< vector< int > > rand_rec_axis_split_maze(int nbLines, int nbCols, float steadiness);
    • vector< vector< int > > rand_kruskal_maze(int nbLines, int nbCols);
    • vector< vector< int > > rand_prim_maze(int nbLines, int nbCols,int iLineStart, int iColStart);
    • vector< vector< int > > rand_dfs_maze(int nbLines, int nbCols,int iLineStart, int iColStart);
    • vector< vector< int > > rand_bfs_maze(int nbLines, int nbCols,int iLineStart, int iColStart);
    • vector< vector< int > > rand_dijkstra_maze(int nbLines, int nbCols,int iLineStart, int iColStart);
    • vector< vector< int > > rand_hunt_and_kill_maze(int nbLines, int nbCols,int iLineStart, int iColStart);
    • vector< vector< int > > rand_binary_maze(int nbLines, int nbCols);
    • vector< vector< int > > rand_eller_maze(int nbLines, int nbCols, float mergeProba);
    • vector< vector< int > > rand_sidewinder_maze(int nbLines, int nbCols, float verticalProba);

  • Pour la génération de labyrinthes de formes particulières :
    • vector< vector< int > > snake_maze(int nbLines, int nbCols);
    • vector< vector< int > > spiral_maze(int nbLines, int nbCols);

  • Pour transformer un labyrinthe acyclique connexe en un labyrinthe de taille double correspondant à un parcours eulérien de la forêt du premier :
    • vector< vector< int > > unicycle_from_acyclic_maze(vector< vector< int > > maze);

  • Pour modifier un labyrinthe par ajout/suppression de murs :
    • vector< vector< int > > add_walls_to_maze(vector< vector< int > > maze, int nbWalls);
    • vector< vector< int > > remove_walls_from_maze(vector< vector< int > > maze, int nbWalls);

  • Pour la conversion d'un labyrinthe avec des murs vers un labyrinthe avec des cases libres et des cases interdites :
    • vector< vector< bool > > convert_to_block_maze(vector< vector > maze);

Format du résultat

Toutes les fonctions de la lib, sauf convert_to_block_maze, renvoient un vector< vector< int > > représentant le labyrinthe généré. Si on appelle res ce résultat, alors:

  • il y a un mur entre la case (iLine,iCol) et la case (iLine,iCol+1) lorsque (res[iLine][iCol]&1)==0
  • il y a un mur entre la case (iLine,iCol) et la case (iLine+1,iCol) lorsque (res[iLine][iCol]&2)==0

Remarque : Les murs séparant la grille de l'extérieur ne sont pas indiqués, seuls ceux séparant deux cases de la grille le sont. Il faut également noter que l'opérateur & n'est pas prioritaire sur l'opérateur ==.

Compilation

Pour utiliser cette librairie, il faut avoir accès au svn tasks. Il suffit alors d'inclure un header : 

#include "rand_generators/rand.h"

et de rajouter "-I$FIOI_COMMON" dans la ligne de compilation :
g++ -o exe monGenerateur.cpp -I$FIOI_COMMON

Note : Si vous utilisez le svn, vous avez probablement défini la variable d'environnement $FIOI_COMMON qui pointe vers le dossier tasks/v01/_common du svn. Si ce n'est pas le cas, définissez cette variable d'environnement.

Détails des algorithmes

rand_maze

vector< vector > rand_maze(int nbLines, int nbCols);

Renvoie un labyrinthe acyclique connexe de façon uniforme en utilisant l'algorithme de Wilson. Peut être assez lent pour de grandes tailles de grille. Plus d'informations ici.

rand_rec_split_maze

vector< vector > rand_rec_split_maze(int nbLines, int nbCols);

Renvoie un labyrinthe acyclique connexe généré selon un procédé récursif partant d'une grille sans aucun mur : On partitionne la zone courante en deux connexes (par l'utilisation de deux BFS concurrents lancés depuis deux graines aléatoires), on construit la frontière entre les deux zones en ne laissant qu'un seul passage et on rappelle récursivement sur chacune des deux zones. Plus d'informations ici.

rand_rec_axis_split_maze

vector< vector > rand_rec_axis_split_maze(int nbLines, int nbCols, float steadiness);

Renvoie un labyrinthe acyclique connexe généré selon un procédé récursif partant d'une grille sans aucun mur : On partitionne la zone rectangulaire courante en deux rectangles, on construit la frontière entre les deux zones en ne laissant qu'un seul passage et on rappelle récursivement sur chacun des deux sous-rectangles. La direction de coupure est choisie uniformément entre les deux orientations possibles. Le choix de l'indice de coupure dépend du paramètre steadiness dont la valeur doit être entre 0.0 et 1.0. En notant s ce paramètre, le ratio de coupe sera tiré uniformément dans [s/2,1-s/2]. Mettre ce paramêtre à 1.0, entraîne donc que l'on va couper à chaque fois en deux parts quasi-égales, tandis que le mettre à 0.0 correspond à choisir l'indice de coupure uniformément parmi tous les indices possibles. Plus d'informations ici.

rand_kruskal_maze

vector< vector > rand_kruskal_maze(int nbLines, int nbCols);

Renvoie un labyrinthe acyclique connexe obtenu comme un arbre couvrant de la grille en utisant une variante de l'algorithme de Kruskal où les arêtes sont prises dans un ordre aléatoire. Plus d'informations ici.

rand_prim_maze

vector< vector > rand_prim_maze(int nbLines, int nbCols,int iLineStart, int iColStart);

Renvoie un labyrinthe acyclique connexe obtenu comme l'arbre de découverte d'un parcours partant d'une case de départ où à chaque étape on choisit uniformément un sommet de l'ensemble actif. Plus d'informations ici.

rand_dfs_maze

vector< vector > rand_dfs_maze(int nbLines, int nbCols,int iLineStart, int iColStart);

Renvoie un labyrinthe acyclique connexe obtenu comme l'arbre de découverte d'un parcours en profondeur partant d'une case de départ où (l'ordre d'énumération des voisins est randomisé). Plus d'informations ici.

rand_bfs_maze

vector< vector > rand_bfs_maze(int nbLines, int nbCols,int iLineStart, int iColStart);

Renvoie un labyrinthe acyclique connexe obtenu comme l'arbre de découverte d'un parcours en largeur partant d'une case de départ (l'ordre d'énumération des voisins est randomisé).

rand_dijkstra_maze

vector< vector > rand_dijkstra_maze(int nbLines, int nbCols,int iLineStart, int iColStart);

Renvoie un labyrinthe acyclique connexe obtenu comme l'arbre de découverte d'un Dijkstra partant d'une case de départ, les coûts des arêtes étant tirés uniformément dans l'intervalle [0,1].

rand_hunt_and_kill_maze

vector< vector > rand_hunt_and_kill_maze(int nbLines, int nbCols,int iLineStart, int iColStart);

Renvoie un labyrinthe acyclique connexe obtenu comme l'arbre de découverte d'un parcours "hunt-and-kill" comme décrit ici.

rand_binary_maze

vector< vector > rand_binary_maze(int nbLines, int nbCols);

Renvoie un labyrinthe acyclique connexe obtenu selon le procédé suivant : On part de la grille avec tous les murs possibles et, pour chaque case, on décide de briser soit le mur à sa droite, soit celui en dessous (de façon équiprobable sauf si l'un des deux choix est impossible). Plus d'informations ici.
